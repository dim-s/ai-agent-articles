# AI Agent Articles

Публичный сайт статей о том, как строить автономных ИИ-агентов под разработку продуктов.

разрешены при **обязательной ссылке на автора и на этот сайт**.

Сайт собран на [Hugo](https://gohugo.io/) (один бинарь, без npm-зависимостей),
публикуется через GitHub Pages. Дизайн — минимализм, фокус на чтении.

## Про префикс `dmitry-*`

В статьях встречаются агенты с префиксом `dmitry-*` (например `dmitry-manager`,
`dmitry-dreamer`). **Это просто имя автора** — вся система списана лично с него, поэтому и
названа его именем. Вы можете списать аналогичную систему **с себя** и назвать её как
угодно: со своим префиксом или вовсе без него. Префикс не несёт технического смысла, это
лишь маркер «это персональный слепок конкретного человека».

## Структура

```
content/
  ru/                       # русские статьи (сейчас единственный язык)
    _index.md               # главная
    about.md                # о проекте, лицензия, про dmitry-*
    articles/
      <slug>/
        index.md            # статья (leaf bundle)
        *.svg, *.png        # иллюстрации рядом со статьёй
  en/                       # задел на английский (пока скрыт: languages.en.disabled)
layouts/                    # шаблоны (минимализм)
assets/css/main.css         # стили
i18n/                       # строки интерфейса (ru.toml, en.toml)
```

Двуязычность заложена структурно: добавить английскую версию — это
`content/en/articles/<slug>/index.md` и снятие `disabled` у `languages.en` в `hugo.toml`.

## Локально

```sh
hugo server           # превью на http://localhost:1313
hugo                  # сборка в ./public
```

## Добавить статью

1. `content/ru/articles/<slug>/index.md` с front-matter (`title`, `date`, `summary`).
2. Иллюстрации — в ту же папку, ссылки относительные (`![alt](image.png)`).
3. `hugo server` — проверить, и закоммитить.
